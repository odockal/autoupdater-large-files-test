# autoUpdater Large File Test

Reproduces the macOS crash that occurs when Electron's `autoUpdater` downloads an
update package larger than 1 GB (Squirrel.Mac integer-overflow / truncation bug).

## How it works

| Step | What happens |
|------|-------------|
| Build v1 | Small app — the installed baseline version |
| Build v2 | Same app + a ~1.1 GB zero-padded resource file |
| Serve v2 | `python3 -m http.server` serves the v2 zip and a JSON manifest |
| Trigger update | Run v1, click **Check for Updates** — it fetches the manifest and downloads the v2 zip |
| Crash | macOS Squirrel.Mac crashes (or silently fails) because the zip exceeds 1 GB |

## Prerequisites

- **macOS on Apple Silicon** (arm64; the bug is macOS-specific, Squirrel.Mac is only used on macOS)
- **Node.js 24+** and **npm**
- **Python 3** (`python3 -m http.server`)
- A valid **Apple Developer signing identity** for `codesign`
  (Squirrel.Mac rejects unsigned apps; see _Code Signing_ below)

## Quick start

All scripts live in `scripts/` and must be run from the project root on macOS.

### 1. Build v1 (the initial, small version)

```bash
bash scripts/01-build-v1.sh
```

This installs npm dependencies and produces:

```
out/make/AutoUpdaterTest-1.0.0-darwin-arm64.zip
```

**Install v1** by unzipping and dragging `AutoUpdaterTest.app` to `/Applications`
(or run it directly from the `out/` directory for testing).

### 2. Build v2 (with the 1.1 GB resource)

```bash
bash scripts/02-build-v2.sh
```

This:
1. Creates `resources/large-file.bin` (~1.1 GB of zeros)
2. Rebuilds the app at version 2.0.0, embedding that file via `extraResource`
3. Produces a zip well over 1 GB

### 3. Set up the update server

```bash
bash scripts/03-setup-server.sh
```

Copies the v2 zip to `update-server/releases/latest.zip` and writes
`update-server/updates/latest.json`:

```json
{
  "url": "http://localhost:8000/releases/latest.zip",
  "name": "2.0.0",
  "notes": "Version 2.0.0 with large payload.",
  "pub_date": "…"
}
```

### 4. Start the local update server

```bash
bash scripts/04-start-server.sh
```

Starts `python3 -m http.server 8000` from the `update-server/` directory.
Keep this terminal open.

### 5. Trigger the update from v1

1. Launch the **v1** app (`AutoUpdaterTest.app` — version 1.0.0).
2. Click **Check for Updates**.
3. The app contacts `http://localhost:8000/updates/latest.json`, receives the
   manifest, and begins downloading the 1.1 GB zip.
4. On affected macOS / Electron versions the app crashes or silently hangs
   during download.

## Code signing

`autoUpdater` on macOS uses **Squirrel.Mac**, which requires the app to be
code-signed with a trusted Apple Developer identity.

### Option A — Developer ID (recommended for accurate reproduction)

```bash
export CODESIGN_IDENTITY="Developer ID Application: Your Name (TEAMID)"
bash scripts/01-build-v1.sh
bash scripts/02-build-v2.sh
```

### Option B — Ad-hoc signing (quick local test)

Ad-hoc signing lets the app launch, but Squirrel.Mac may still reject the
unsigned update during `quitAndInstall`. Useful for testing the _download_
crash before the install phase:

```bash
# After packaging, ad-hoc sign the .app:
codesign --deep --force --sign - out/AutoUpdaterTest-darwin-arm64/AutoUpdaterTest.app
```

### Option C — Disable Gatekeeper (for isolated VMs only)

```bash
sudo spctl --master-disable   # re-enable afterwards!
```

```
│   ├── 03-setup-server.sh   # Copy zip + write JSON manifest
│   └── 04-start-server.sh   # python3 -m http.server 8000
├── resources/
│   └── large-file.bin       # Created by 02-build-v2.sh (gitignored)
└── update-server/
    ├── releases/            # large-file.zip goes here (gitignored)
    └── updates/
        └── latest.json      # Generated by 03-setup-server.sh (gitignored)
```

## Adjusting the file size

Edit `FILE_SIZE_MB` in `scripts/02-build-v2.sh`:

```bash
FILE_SIZE_MB=1126  # ~1.1 GB; minimum to trigger the bug is ~1024 (1 GB)
```

## Feed URL

The feed URL is set in `src/main.js`:

```js
const UPDATE_SERVER = 'http://localhost:8000'
const FEED_URL = `${UPDATE_SERVER}/updates/latest.json`
```

Change `UPDATE_SERVER` if you run the server on a different host or port.
